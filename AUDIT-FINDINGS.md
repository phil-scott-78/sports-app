## CRITICAL

### C1. Tennis is unusable: set scores dropped + only one match per tournament shows
**Symptom:** Finished tennis matches show two names + "Final" with no scoreline anywhere; the detail "Sets" panel never appears. Separately, each tournament renders as a single card showing only one (the first) of 50–110 matches; live count and sort are blind to the rest.
**Root cause (two linked bugs):**
- ESPN tennis `linescores` have `{value, winner}` with **no `period` field**, and no top-level `competitor.score`. `normalize.js:103` filters `ls.period != null` → every set dropped → `periodScores:[]` and `score:undefined`.
- ESPN models a whole tournament as ONE event with matches nested in `groupings[].competitions[]`; the worker correctly flattens to 54–110 competitions, but the app renders only `event.main` (`competitions.first`).
**Files:** `worker/src/normalize.js` (the `ls.period != null` filter); `app/lib/src/ui/scores_page.dart` (leagueSections one-card-per-event, `_LiveCountChip`, `_byStatusThenTime`); `app/lib/src/models.dart` (`SportEvent.main`); `app/lib/src/ui/game_detail_page.dart` (tennis detail-tap uses `event.main`).
**Fix:**
- Worker: keep linescores lacking `period` and synthesize `period: ls.period ?? idx+1` (regression-safe — every other sport already supplies `period`). This revives the detail "Sets" SetStrip.
- App: expand multi-competition tennis events to one card per competition; make live-count/sort/detail-tap operate per-competition. Gate on a **data-driven flag** (or scoreKind=numeric), NOT layout==headToHead alone — MMA shares that shape and would be exploded into per-bout rows unintentionally.
- The list big-number is still blank after the period fix (tennis has no `c.score`) — derive a sets-won tally or render a SetStrip-style mini-line (see U6).
**Blast radius:** Worker filter change is tennis-only in effect; the app expansion logic must avoid catching MMA.

### C2. Last/Next favorite scores render "[object Object]" for EVERY team sport
**Symptom:** Favorites rail "Last" row shows `[object Object]–[object Object]` (W/L/D letter and opponent correct, numbers garbage) — NBA, MLB, NHL, soccer.
**Root cause:** The team-**schedule** endpoint serializes `competitor.score` as an OBJECT (`{value,displayValue}`, soccer adds `$ref/winner/source`), unlike the scoreboard (string). `buildScore` does `String(raw)` → `[object Object]` and `intOrNull(raw)` → NaN (value also lost).
**Files:** `worker/src/normalize.js` (`buildScore`); renders faithfully in `app/lib/src/ui/favorite_team_card.dart` — do NOT fix app-side.
**Fix:** At the top of `buildScore`, coerce object → scalar before stringify: `if (typeof raw === 'object') raw = raw.displayValue ?? raw.value ?? ''`. Scoreboard strings untouched (byte-identical).
**Blast radius:** Single chokepoint repairs all team sports at once. Same root cause as the soccer-club finding and the favorites finding — **one fix.**

**Blast radius:** Club leagues unaffected (their schedule populates). Same root direction also fixes the permanently-dead F1-constructor card (see H-section).
